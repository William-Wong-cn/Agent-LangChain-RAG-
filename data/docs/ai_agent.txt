AIAgent（Artificial Intelligence Agent，人工智能代理）
是一种以大语言模型（LLM）为核心的智能系统，
其目标不是单次问答，而是完成一个明确的任务目标。

与传统聊天模型不同，AIAgent 具备“自主性”，
能够根据任务需求进行推理、规划并采取行动。
在执行过程中，Agent 可以调用外部工具，
并根据工具返回的结果不断调整后续决策。

一个完整的 AIAgent 系统通常包含以下核心组成部分：

1. 大语言模型（LLM）
LLM 是 Agent 的“决策大脑”，负责理解用户意图、
进行推理、生成计划以及整合最终答案。

2. 工具系统（Tools）
工具用于扩展 Agent 的能力边界，例如：
- 本地知识库检索（RAG）
- 在线搜索引擎
- 文件读写
- 代码执行或计算工具
Agent 会根据问题类型自主选择是否调用工具。

3. 记忆机制（Memory）
记忆通常分为两类：
- 短期记忆：当前对话上下文
- 长期记忆：以向量数据库形式存储的知识库（如 FAISS）
长期记忆使 Agent 能够基于已有文档进行语义检索。

4. 规划与执行（Planning & Execution）
Agent 可以将复杂问题拆解为多个子任务，
按照“思考 → 行动 → 观察 → 再思考”的循环方式执行，
这一过程常被称为 ReAct 模式。

5. 反馈与循环（Loop）
Agent 会根据工具返回的结果判断任务是否完成，
若未完成，则继续规划并执行新的动作。

在工程实践中，常见的 Agent 架构包括：
- 单 Agent 多工具架构
- 多 Agent 协作架构（如 Planner / Researcher / Writer）

多 Agent 架构通过职责拆分，
提升了系统的可解释性、稳定性和可扩展性，
在复杂问答、知识检索和自动化任务中具有明显优势。

AIAgent + RAG 是当前主流的应用模式之一。
通过将本地文档构建为向量知识库，
Agent 可以优先从可信的本地资料中获取答案；
当本地知识不足时，
再通过在线搜索作为兜底信息来源，
从而在准确性与时效性之间取得平衡。
